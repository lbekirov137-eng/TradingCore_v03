# QUALITY GATE

Любое изменение стратегии должно пройти:

1. Код собирается без ошибок.

2. Unit Test проходит.

3. Backtest не хуже предыдущей версии.

4. Profit Factor не уменьшается.

5. Max Drawdown не увеличивается.

6. Win Rate не ухудшается.

7. Изменение документируется.

Только после этого изменение считается принятым.
